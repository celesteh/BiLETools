NetworkGui a class that draws nice GUIs for BiLE



Requires:

•  ProcessTools, a quark

•  OscGroups, external program, download from http://www.audiomulch.com/~rossb/code/oscgroups/

•  OscGroupClient, Les's altered version

•  NetAPI


See also: SharedCV


Usage


This class handles a GUI for shared and local variables.  It's quite similar to Conductors.


The main constructor class is make. It's first argument should be an instance of NetAPI.  The second argument should be an array listing all local (as in, non-shared) values. If there are no non-shared values, you should pass in an empty array.  The third argument is a function.


The function has as many arguments as you'd like. The first one is the instance of NetworkGui cirrently being created. Then come all the values you'd like sliders for. Any that you've listed in the local array will only be created locally. The rest will be shared.


(


var api, matrix;

api = NetAPI.broadcast("Nick");



matrix = NetworkGui.make(api, [\amp], {|gui,amp, freq|

gui.synth_(

(

instrument: \default

),

(

amp: amp,

freq:freq

)

);


}).show;

)


Boot the server and press the play button to play.


Any argument names that are also ControlSpec keys will be automagically mapped to the correct ControlSpec.  You can also explicitly set the spec.


(


var api, matrix;

api = NetAPI.broadcast("Nick");



matrix = NetworkGui.make(api, [\amp], {|gui,amp, freq|

amp.spec_(\amp);

freq.spec_(\freq);

gui.synth_(

(

instrument: \default

),

(

amp: amp,

freq:freq

)

);


}).show;

)


It's also possible to specify functions for the synth parameters.


(


var api, matrix;

api = NetAPI.broadcast("Nick");



matrix = NetworkGui.make(api, [\db], {|gui, db, freq|

gui.synth_(

(

instrument: \default

),

(

amp: [db, {|db| db.dbamp}],

freq:freq

)

);


}).show;

)

For that we used an array. The first item is the SharedCV we're using and then comes a function. The return value of the function gets passed to the synth.




You can specify that you'd rather use knobs than sliders:


(


var api, matrix;

api = NetAPI.broadcast("Nick");



matrix = NetworkGui.make(api, [\amp], {|gui,amp, freq|

amp.guitype_(\knob);

freq.guitype_(\knob);

gui.synth_(

(

instrument: \default

),

(

amp: amp,

freq:freq

)

);


}).show;

)


You can add extra sliders later:


(


var api, matrix;

api = NetAPI.broadcast("Nick");



matrix = NetworkGui.make(api, [\amp], {|gui,amp, freq|

gui.name = "Demo";

gui.synth_(

(

instrument: \default

),

(

amp: amp,

freq:freq

)

);


}).show;

matrix.addLocal(\bipolar);

matrix.addShared(\rq);

)


You can also add this on to already existing windows:


(


var api, chat, matrix, win;


api = NetAPI.broadcast("Nick");

chat = BileChat(api);

matrix = NetworkGui.make(api, [\amp], {|gui, amp, freq|

gui.synth_(

(

instrument: \default

),

(

amp: amp,

freq:freq

)

);


}).show(chat.win);

)


It's possible to mount remote shared data in two ways. One is to assume that every other user is sharing the same data as you and thus will create sliders identitcal to your own shared sliders for every user on the network:


(


var api, chat, matrix, win;


api = NetAPI.broadcast("Nick");

chat = BileChat(api);

matrix = NetworkGui.make(api, [\amp], {|gui, amp, freq|

gui.synth_(

(

instrument: \default

),

(

amp: amp,

freq:freq

)

);

gui.copyRemote = true;


}).show(chat.win);

)



The other way to mount remote shared data is to mount everything that is advertised. An attempt is made to guess the right control spec based on the shared item's name.


(


var api, chat, matrix, win;


api = NetAPI.broadcast("Nick");

chat = BileChat(api);

matrix = NetworkGui.make(api, [\amp], {|gui, amp, freq|

gui.synth_(

(

instrument: \default

),

(

amp: amp,

freq:freq

)

);

gui.mountRemote = true;


}).show(chat.win);

)



You can add specific remote shared data. It's also possible to specify a pattern, rather than (or in addition to) a synth.



(


var api, chat, matrix, win;


api = NetAPI.broadcast("Nick");

chat = BileChat(api);

matrix = NetworkGui.make(api, [\amp], {|gui, amp, freq|

var dur;


dur = gui.addRemote(\dur);

dur.sp(1, 0.1, 5, 0, \lin, gui); //default, min, max, step size, warp, changer

gui.pattern_(

Pbind(

\freq, freq,

\amp, amp,

\dur, dur

)

);


}).show(chat.win);

)
